# v3.6.10 변경 사항 (2026-05-13)

## 핵심 기능 — 공문 양식 빌더 완성

v3.6.0 ~ v3.6.10 누적으로 공문 양식 빌더가 양식 슬롯 한계를 넘어 사용자
콘텐츠에 맞춰 자유롭게 빌드되도록 발전. 한글2018 / 한컴뷰어 양 환경에서
시각 동등 보장.

### 사용 방법 변경 — `examples/example_values_gongmun.json` 참고

공문 빌드 시 values.json 에 다음 7개 위계 키를 자유 분량으로 입력 가능:

```json
{
  "수신자": "내부 임직원",
  "text_005": "(경유)",
  "text_006": "제목",
  "본문":      ["1. ...", "2. ...", "3. ...", "4. ..."],
  "본문_가나": ["가. ...", "나. ...", "다. ..."],
  "본문_1)":  ["1) ...", "2) ..."],
  "본문_가)":  [],
  "본문_(1)":  [],
  "본문_①":   ["① ...", "② ..."],
  "붙임":      ["붙임 1. ...", "2. ...", "3. ..."]
}
```

- 각 위계 배열의 항목 수에 따라 단락이 자동 확장
- 빈 배열 또는 미입력 시 해당 위계 단락 자체가 출력에서 사라짐
- 표 구조(헤더/발신부) 단락은 hp:tbl 보존 예외로 무조건 유지
- `수신자` 키만 입력하면 "수신" 라벨은 자동 부여 (text_004 직접 입력 불필요)
