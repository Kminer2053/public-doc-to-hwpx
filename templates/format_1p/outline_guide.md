# 1페이지 보고서 목차 구성 가이드라인

> 이 가이드는 1페이지 보고서(`format_1p`) 양식 사용 시, 보고 목적에 맞는 **목차 구조**를 결정하기 위한 표준 안내입니다.
> 양식의 4개 섹션 슬롯(`text_004`~`text_023`)에 어떤 제목·내용을 넣을지 정할 때 활용합니다.

---

## 1. 기본 전제

공공기관 1페이지 보고서는 문서 양식보다 **보고받는 사람이 무엇을 해야 하는지**를 기준으로 목차를 정해야 합니다.

| 보고받는 사람이 해야 할 일 | 보고 유형 |
|---|---|
| 알면 됨 | 현황·동향 보고 |
| 판단·선택해야 함 | 검토·의사결정 보고 |
| 실행 승인이 필요함 | 사업·정책 계획 보고 |
| 완료 사실·성과 확인 | 결과·성과 보고 |
| 후속조치가 필요함 | 회의결과·이슈 보고 |

권장 구조는 **제목 + 보고요지 + 본문 3~5절** 입니다.

---

## 2. 양식 슬롯과 목차의 대응

`format_1p/skeleton.hwpx` 의 본문 영역은 **4개 섹션** 으로 구성되어 있습니다 (각 섹션 = □ 헤더 + ◦ 항목 2 + 세부 1 + * 주석 1).

| 양식 슬롯 그룹 | 4절 목차 매핑 | 5절 목차 매핑 (압축) | 3절 목차 매핑 (확장) |
|---|---|---|---|
| **섹션 1** (text_004~008) | 보고요지 | 보고요지 (한 줄 결론) | 개요 |
| **섹션 2** (text_009~013) | 배경·현황 | 추진배경 + 현황·문제점 합침 | 주요내용 (확장) |
| **섹션 3** (text_014~018) | 주요내용·검토 | 주요내용·대응방안 | 주요내용 (계속) |
| **섹션 4** (text_019~023) | 향후계획·요청사항 | 향후계획·요청사항 | 향후계획 |
| **붙임** (장01~본문_주석_002) | 세부 일정·후속조치 | (동일) | (동일) |

> 양식은 본문 4섹션을 가지고 있으므로, **4절 기본형이 가장 자연스럽습니다.**
> 5절이 필요하면 한 섹션에 두 절을 합치거나, 압축형(3절) 이면 마지막 섹션을 비워둡니다.

## 3. 공통 골격

### 3.1 4절 기본형 (양식 기본 매칭)

| 구분 | 표준 목차 | 핵심 질문 |
|---|---|---|
| 1 | 보고요지 | 그래서 결론이 무엇인가? |
| 2 | 배경·현황 | 왜 지금 보고하는가? |
| 3 | 주요내용·검토 | 무엇을 봐야 하는가? |
| 4 | 향후계획·요청사항 | 무엇을 결정·조치해야 하는가? |

### 3.2 5절 확장형

| 구분 | 표준 목차 | 핵심 질문 |
|---|---|---|
| 1 | 보고요지 | 한 줄 결론은 무엇인가? |
| 2 | 추진배경 | 왜 필요한가? |
| 3 | 현황·문제점 | 현재 무엇이 문제인가? |
| 4 | 주요내용·대응방안 | 어떻게 할 것인가? |
| 5 | 향후계획·요청사항 | 다음 조치는 무엇인가? |

### 3.3 3절 압축형

| 구분 | 표준 목차 | 사용 상황 |
|---|---|---|
| 1 | 개요 | 사안의 핵심만 전달 |
| 2 | 주요내용 | 현황·검토·방안 통합 |
| 3 | 향후계획 | 일정·조치·요청사항 정리 |

## 4. 보고목적별 표준 목차

### 4.1 현황·동향 보고
**목적:** 특정 사안의 현재 상태와 흐름을 빠르게 공유

| 순서 | 표준 목차 | 작성 포인트 |
|---|---|---|
| 1 | 보고요지 | 핵심 변화, 중요 수치, 판단 포인트 |
| 2 | 주요현황 | 수치, 사실, 비교 기준 중심 |
| 3 | 주요동향 | 외부환경, 이해관계자, 언론·시장 흐름 |
| 4 | 영향·시사점 | 기관 업무에 미칠 영향 |
| 5 | 향후관리 | 모니터링 계획, 추가 보고 시점 |

**압축형:** `현황 → 시사점 → 향후관리`

### 4.2 검토·의사결정 보고
**목적:** 상급자가 선택할 수 있도록 대안과 판단근거 제시

| 순서 | 표준 목차 | 작성 포인트 |
|---|---|---|
| 1 | 검토결과 | 권고안·결론을 먼저 제시 |
| 2 | 검토배경 | 왜 검토가 필요한지 |
| 3 | 쟁점사항 | 법령, 예산, 이해관계, 리스크 |
| 4 | 대안검토 | 대안별 장단점·비용·효과 비교 |
| 5 | 결정요청 | 승인·선택·지시가 필요한 사항 |

**압축형:** `검토배경 → 대안검토 → 결정요청`

### 4.3 사업·정책 계획 보고
**목적:** 신규 사업·제도·프로젝트의 추진 승인·방향 설정

| 순서 | 표준 목차 | 작성 포인트 |
|---|---|---|
| 1 | 추진방향 | 사업의 핵심 목표와 방향 |
| 2 | 추진배경 | 필요성, 정책환경, 기관전략 연계 |
| 3 | 주요내용 | 과업 범위, 대상, 세부 추진내용 |
| 4 | 추진체계 | 일정, 예산, 역할분담, 협업체계 |
| 5 | 향후계획 | 세부 일정, 보고·결재·협조사항 |

**압축형:** `추진배경 → 주요내용 → 향후계획`

### 4.4 문제해결·개선방안 보고
**목적:** 현재 문제를 해결하거나 업무방식을 개선하기 위한 방안 제시

| 순서 | 표준 목차 | 작성 포인트 |
|---|---|---|
| 1 | 개선방향 | 핵심 개선안 요약 |
| 2 | 현황·문제점 | 문제 발생 구조와 근거 데이터 |
| 3 | 원인분석 | 제도·프로세스·인력·시스템 원인 |
| 4 | 개선방안 | 단기·중기·장기 또는 내부·외부 구분 |
| 5 | 실행계획 | 담당, 일정, 비용, 기대효과 |

**압축형:** `문제점 → 개선방안 → 실행계획`

### 4.5 이슈·리스크·사고 보고
**목적:** 돌발 상황·민원·언론보도·사고·리스크 신속 대응

| 순서 | 표준 목차 | 작성 포인트 |
|---|---|---|
| 1 | 상황요약 | 발생 사실, 현재 상태, 긴급도 |
| 2 | 발생경위 | 시간순 경과, 확인된 사실 중심 |

| 4 | 조치현황 | 이미 한 조치 + 진행 중 조치 |
| 5 | 향후대응 | 추가 조치·보고라인·재발방지 |

**압축형:** `상황요약 → 조치현황 → 향후대응`

## 5. 작성 규칙 (필수)

1. **목차는 3~5절**로 제한합니다.
2. **보고요지·검토결과를 상단(섹션 1)** 에 배치합니다.
3. 현황과 문제점이 겹치면 **현황·문제점** 으로 합칩니다.
4. 표·도식은 1개만 핵심적으로 사용합니다.
5. 본문은 판단에 필요한 내용만 두고, 세부자료는 **붙임·참고** 로 뺍니다.
6. 마지막 절은 반드시 **향후계획·조치사항·결정요청** 중 하나로 마무리합니다.

## 6. 빌드 시 활용 방법

1. 보고 목적을 먼저 결정 → 위 11개 유형 중 선택
2. 매칭되는 표준 목차 (4절·5절·3절) 선정
3. 양식 슬롯에 매핑:
   - 섹션 1 (text_004 □ + text_005·006 ◦ + text_007 - + text_008 *) ← 첫 번째 절
   - 섹션 2 (text_009 □ + text_010·011 ◦ + text_012 - + text_013 *) ← 두 번째 절
   - 섹션 3 (text_014 □ + text_015·016 ◦ + text_017 - + text_018 *) ← 세 번째 절
   - 섹션 4 (text_019 □ + text_020·021 ◦ + text_022 - + text_023 *) ← 네 번째 절
   - 5절 확장이 필요하면 한 섹션에 두 절 압축
   - 3절 압축은 마지막 섹션을 비워둠
4. 붙임 영역 (장01·02 + 본문_절_001·002) 에 세부 일정·후속조치 매핑
5. `python scripts/fill_skeleton.py` 로 빌드

## 7. 작성 예시

### 예시 1: 사업·정책 계획 보고 (4절형)
- 섹션 1 **추진방향** — "AX 시대 인적자원 혁신을 위한 전사적 AI역량 강화"
- 섹션 2 **추진배경·필요성** — 글로벌 AI 도입 89% vs 70% 좌초, 인적역량 부족
- 섹션 3 **주요내용** — 3대 KPI (챔피언 20% / 예산 25% / 1:5 비율)
- 섹션 4 **향후계획** — 18개월 3단계 + 후속 조치
- 붙임 — 풀버전 추진계획 별도 보고 / 챔피언 추천 협조

### 예시 2: 이슈·리스크 보고 (압축형 3절)
- 섹션 1 **상황요약** — 발생 사실·긴급도
- 섹션 2 **조치현황** — 이미 한 조치 + 진행 중 조치
- 섹션 3 **향후대응** — 추가 조치·재발방지
- 섹션 4 — **빈 값** 처리

### 예시 3: 검토·의사결정 보고 (4절형)
- 섹션 1 **검토결과** — 권고안 또는 결론을 먼저
- 섹션 2 **검토배경·쟁점** — 왜 검토가 필요한지 + 법령·예산·리스크
- 섹션 3 **대안검토** — 대안별 장단점 비교
- 섹션 4 **결정요청** — 승인·선택·지시가 필요한 사항
