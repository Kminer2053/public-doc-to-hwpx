# 양식 선택 가이드 (Format Selection)

> SKILL.md 워크플로우 ② 단계에서 사용하는 양식 결정 기준.
> 참조 hwpx/hwtx 파일이 없을 때 4개 옵션 중 자동/수동 선택.

---

## 1. 4개 양식 한 눈에 보기

| 양식 | 분량 | 독자 | 작성 방식 | 글머리 위계 | 핵심 특징 |
|------|------|------|----------|-------------|----------|
| **1p 보고서** | A4 1쪽 (강제) | 의사결정자(상급자) | 개조식 | □ ○ - * | 두괄식 음영 박스 + 핵심 항목만 |
| **풀버전 보고서** | A4 5–30쪽 | 상급자·관계부서 | 개조식 + 서술식 혼용 | Ⅰ. 1. 가. (1) | 표지 + 목차 + 보고요약 + 본문 + 별첨 |
| **시행문** | A4 1–3쪽 | 타부서·외부기관·일반국민 | 서술식 + 개조식 혼용 | 1. 가. 1) (1) | 수신·제목·본문·발신명의 형식 엄수 |
| **이메일** (텍스트) | 약 200–500자 | 내·외부 협업자 | 개조식 단순화 | □ - * | 제목 첫째, 결론 둘째, 기한 셋째 |

## 2. 양식 자동 추천 결정트리

콘텐츠 입력을 분석해 다음 순서로 양식 추천:

```
[입력 분석]
  │
  ├─ 사용자가 명시적으로 양식 지정?
  │     예 → 그대로 진행
  │     아니오 ↓
  │
  ├─ "보내" "발송" "통보" "공지" "지시" 키워드 + 외부 수신자?
  │     예 → 시행문 추천
  │     아니오 ↓
  │
  ├─ "메일" "이메일" "회신" 키워드 + 짧은 분량?
  │     예 → 이메일 추천
  │     아니오 ↓
  │
  ├─ 콘텐츠 추정 분량(개조식 변환 후) 산출
  │     ≤ 25 줄 → 1p 보고서 추천
  │     > 25 줄 → 풀버전 보고서 추천
  │
  └─ 사용자에게 최종 확인 (옵션 1개 + 대안 1개 제시)
```

분량 추정은 `layout_optimizer.estimate_lines()` 함수가 담당. 페이지당 38–42줄 가정.

## 3. 양식별 판별 키워드

### 3-1. 시행문 신호

- 동사: 알리다, 통보하다, 지시하다, 협조하다, 요청하다, 공지하다, 발송하다
- 명사: 안내, 통지, 시행, 공고, 회신 요청, 협조 요청
- 수신: `수신: ~`, `귀 부서`, `귀 기관`, `각 부서장`
- 결문: `~하기 바랍니다`, `~하여 주시기 바랍니다`, `~하시기 바랍니다`

### 3-2. 1페이지 보고서 신호

- 명사: 보고, 결과보고, 검토결과, 동향, 현황, 진행 경과
- 분량 한정: `핵심만`, `요약`, `한 장으로`, `간략하게`, `1p`, `1쪽`
- 두괄식 의도: 결정·판단을 빨리 끌어내야 하는 보고

### 3-3. 풀버전 보고서 신호

- 명사: 기획서, 추진계획, 검토보고서, 사업계획서, 분석보고서, 백서
- 분량 신호: 다수의 분석 항목·근거자료·예산표·일정표 동반
- 구조 신호: `Ⅰ. ~ Ⅱ. ~ Ⅲ. ~`처럼 큰 장(章) 구분 필요

### 3-4. 이메일 신호

- 채널 명시: `메일로`, `이메일로`, `~에게 보낼`
- 인사말 필요성: `안녕하세요`, `~님께`
- 구조: 제목 → 인사 → 본론 → 요청·기한 → 마무리

## 4. 사용자 참조 파일이 있을 때

사용자가 hwpx/hwtx 참조 파일을 제공하면, 4개 양식 분기를 거치지 **않고** 다음 워크플로우로:

```
[참조 파일 수신]
  │
  ├─ 1단계: kordoc:detect_format 으로 포맷 확인
  │     hwpx / hwtx 가 아니면 사용자에게 재요청
  │
  ├─ 2단계: kordoc:parse_form 으로 빈칸(레이블·필드) 구조 파악
  │     빈칸 있음 → 양식형 (Form-based) 으로 진행
  │     빈칸 없음 → 서식 추출형 (Style-extract) 으로 진행
  │
  ├─ 3-A 양식형(Form-based):
  │     - 콘텐츠를 빈칸에 매핑 (kordoc:fill_form 사용)
  │     - 자동 매핑 불가 시 사용자에게 매핑표 확인 요청
  │
  └─ 3-B 서식 추출형(Style-extract):
        - hwpx_helpers.extract_secpr_and_colpr 로 서식 추출
        - charPr/paraPr/borderFill ID를 참조 파일과 동일하게 사용
        - 콘텐츠는 4개 양식 중 가장 가까운 구조로 매핑
```

> ⚠️ kordoc:fill_form 결과물도 빌드 후 반드시 `fix_namespaces.py` 실행.

## 5. 양식 변경이 필요한 시점

빌드 진행 중 다음 경우 **양식 변경 권고**:

| 상황 | 현재 양식 | 권고 양식 |
|------|-----------|-----------|
| 1p 양식인데 콘텐츠가 1쪽에 들어가지 않음 (압축 후에도 초과) | 1p 보고서 | 풀버전 보고서 |
| 풀버전인데 핵심만 5분 안에 보고해야 함 | 풀버전 보고서 | 1p 보고서 |
| 시행문인데 분량이 5쪽 넘음 | 시행문 | 시행문 + 별첨(보고서) |
| 이메일인데 분량이 본문 1,000자 초과 | 이메일 | 1p 보고서를 별첨으로 + 이메일은 안내문 |

권고는 사용자에게 옵션으로 제시하고, **자동 변경하지 않습니다**.

## 6. 양식별 빌드 출력 형태

| 양식 | 빌드 출력 | 비고 |
|------|-----------|------|
| 1p 보고서 | `.hwpx` 단일 파일 | 1쪽 강제 |
| 풀버전 보고서 | `.hwpx` 단일 파일 | 표지·목차 자동 페이지 분리 |
| 시행문 | `.hwpx` 단일 파일 | 발신명의 자동 마지막 줄 배치 |
| 이메일 | `.md` 또는 `.txt` (사용자 답변: 텍스트로만) | 본문 복사·붙여넣기용 |

## 7. 결정 우선순위

순서대로 점검:

1. **사용자 명시 지정** — 가장 우선 (`이메일로 작성해줘`, `시행문 양식으로`)
2. **참조 파일 제공** — 두 번째 우선 (참조 파일의 양식 추정)
3. **콘텐츠 분량·키워드 분석** — 결정트리 적용
4. **사용자 최종 확인** — 추천 양식 1개 + 대안 1개 제시
