# 풀버전 보고서 (Full Report) 작성 가이드

> 우수예시: `스마트 편의점 개발 추진계획`(2020.7., 25p), `전문점 운영 편람`(231p)
> 공공기관 보고서 작성 강의자료 + 공공기관 사무관리지침

> **v3.1 기준 빌드 베이스**: `templates/format_full/standard.hwpx` (맑은 고딕, charPr 74개, 이미지 사전 제거됨)가 기본 베이스로 자동 적용됩니다. 표지 대제목용 `COVER`(28pt), `TITLE_XL`(24pt) 등 다양한 위계 스타일이 매핑되어 있습니다.

---

## 1. 핵심 철학

> **표지에서 별첨까지, 의사결정자가 흐름을 이해하고 근거를 검증할 수 있는 완결적 문서.**

분량은 5–30쪽 사이가 표준. 30쪽 넘으면 별책으로 분리 권고.

---

## 2. 표준 골격 (5개 섹션)

```
[1] 표지 (Cover)
    - 회사 로고 + 협조 결재선 + 문서번호·보존기간·보고일자
    - 부제 + 제목 (16pt 굵게)
    - 작성연월 + 작성부서

[2] 목차 (Table of Contents)
    - 「목  차」 가운데 정렬 16pt 굵게
    - Ⅰ. 추진배경 및 목적 ............ 4
      Ⅱ. 추진계획(안) ................ 6
      Ⅲ. 추진일정 ................... 14
      Ⅳ. 행정사항 ................... 15
      【참고자료】
        1. 개발 산출물 예시사례 ......... 15
        2. 스마트화 전환 실사 필요매장 .. 23

[3] 보고내용 요약 (Executive Summary, 1쪽)
    - "보고내용 요약" 가운데 16pt 굵게
    - 핵심 항목별 1–3줄 압축 (배경·계획·일정·예산·협조사항)

[4] 본문 (Body)
    - Ⅰ. → 1. → 가. → (1) → (가) → ① → ㉮ 위계
    - 각 장(章) 시작은 새 페이지

[5] 참고자료/별첨 (Appendix, 선택)
    - 산출물 예시, 통계자료, 관련 법령, 외부자료
```

## 3. 항목별 작성 요령

### 3-1. 표지

공공기관 사무관리지침 기반 표준 표지:

```
┌──────────────────────────────────────────────┐
│ [로고]      ┌─ 결재선 표 ─────────┐            │
│             │ 과장 처장 본부장 대표이사 │            │
│ ┌─ 문서메타 │ 박○○ 박○○ 이○○ 박○○ │            │
│ │ 문서번호  └────────────────┘            │
│ │ 보존기간                                  │
│ │ 보고일자                                  │
│ └────                                      │
│                                              │
│        - 부제(맥락 설명) -                      │
│      제 목 (16pt 굵게, 가운데)                   │
│                                              │
│            2025. 11.                         │
│                                              │
│         부서명 (큰 글씨, 가운데)                  │
└──────────────────────────────────────────────┘
```

### 3-2. 목차

- 제목: 「목  차」 가운데 정렬, 16pt 굵게, 옅은 색 박스(선택)
- 본문: `Ⅰ. 추진배경 및 목적`처럼 **로마숫자 + 마침표 + 두 칸 공백 + 제목**
- 우측에 페이지 번호 (점선·숫자 우측 정렬)
- 참고자료는 「참고자료」로 별도 섹션

### 3-3. 보고내용 요약 (Executive Summary)

**가장 중요한 1쪽**. 의사결정자가 이 한 쪽만 보고도 핵심을 파악해야 함.

공공기관 우수예시(`스마트 편의점 개발 추진계획`) 패턴:

```
▦ 추진배경 및 목적
1. ○○ 사업체계 재구축 전략 이행
2. Post 코로나시대 대응을 위한 비대면 영업체계 구축
3. 핵심분야 기술력 확보를 통한 사업권 유지 및 경쟁력 제고

▦ 추진계획 주요내용
1. 개발방향 : 역구내에 최적화된 자판기형 스마트 편의점
   - 구내영업의 특수성 및 보유 역량 활용차원
2. 개발방법 : 시스템엔지니어링 관점에서 운영체계 개발
   - 단순장비 구매수준이 아니라 스마트화 운영체계 전반을 개발
   - V프로세스 모델에 따른 단계별 산출물을 정확하게 작성
3. 구현 및 테스트
   - 본사 사옥 내 시제품 설치 및 테스트 진행
4. 유연한 조직 및 협업체계 구성
   - 부서별 업무분장 명확화하고 진행단계별 회의실시
   - 세부과업에 필요인력 겸직발령하고 역량평가 보상
5. 운영체계 특허등록 추진
6. 스마트화 전환대상 매장 실사 및 운영계획 구체화

[하단: 3개 정보 박스 - 추진일정 / 소요예산 / 부서별 협조사항]
```

핵심 패턴:
- ▦ 마커가 대제목 (□와 유사)
- 본문은 `1. 2. 3.`으로 시작 (Ⅰ. Ⅱ.가 아니라)
- 하위는 `-` 점선
- 마지막 줄에 3분할 박스로 일정·예산·협조사항 압축

### 3-4. 본문

위계 (체계 B):

```
Ⅰ. 추진배경 (장·章, 18pt 굵게, 새 페이지)
  
  1. 환경분석 (절·節, 14pt 굵게)
  
    가. 외부환경 (목·目, 12pt 굵게)
    
      (1) 거시환경 (12pt 보통)
      
        (가) 정책 동향
        
          ① 세부 항목
          
            ㉮ 더 세부
```

각 장(Ⅰ. Ⅱ. Ⅲ.) 시작은 **새 페이지**로 강제.

### 3-5. 참고자료/별첨

- 「참고자료」 또는 「별첨」 표제 한 페이지에 가운데
- 본문과 페이지 번호 분리 (선택)
- 외부자료는 출처 명기

---

## 4. 분량 가이드

A4, 여백 상30·하25·좌30·우25mm 기준 **약 38–42줄/쪽**.

| 섹션 | 권장 쪽 수 | 비고 |
|------|----------|------|
| 표지 | 1쪽 | 강제 |
| 목차 | 1쪽 | 길어지면 2쪽 가능 |
| 보고내용 요약 | 1쪽 | 강제 |
| 본문 | 5–25쪽 | 장(章) 단위 분리 |
| 참고자료 | 0–10쪽 | 선택 |
| **합계** | **8–38쪽** | |

> 38쪽 초과 시 본문을 2개 권으로 분리하거나, 참고자료를 별책으로 빼는 것을 권고.

## 5. 양식 미준수 위험 신호

- [ ] 표지에 결재선이 없음 → 사용자 확인 후 빈 칸으로라도 삽입
- [ ] 목차의 페이지 번호가 본문과 일치하지 않음 → 빌드 후 자동 갱신
- [ ] 보고내용 요약이 1쪽을 넘김 → 압축
- [ ] 장(章)이 5개 초과 → 통합 검토
- [ ] 한 장(章)이 0.5쪽 미만 → 다음 장과 통합 검토
- [ ] 본문에 `-적/의/것/들` 등 군더더기 표현 → 자동 정리
- [ ] 한 문장 1줄 초과 → 분리

## 6. 빌드 시 hwpx_helpers.py 호출 패턴

```python
# 1. 표지 페이지
parts.extend(make_cover_page_full(
    company_logo='templates/format_full/○○공사_logo.png',
    title='스마트 편의점 개발 추진계획',
    subtitle='- 역구내 환경에 적합한 -',
    date='2025. 11.',
    department='AI혁신처',
    approval_line=[
        ('과장', '박○○'),
        ('처장', '박○○'),
        ('본부장', '이○○'),
        ('대표이사', '박○○'),
    ],
    doc_meta={
        '문서번호': 'AI혁신처-001',
        '보존기간': '준영구 비공개(7)',
        '보고일자': '2025. 11. 5.',
    }
))
parts.append(make_page_break())

# 2. 목차 페이지
parts.extend(make_toc_page(
    title='목  차',
    entries=[
        ('Ⅰ. 추진배경 및 목적', 4),
        ('Ⅱ. 추진계획(안)', 6),
        ('Ⅲ. 추진일정', 14),
        ('Ⅳ. 행정사항', 15),
    ],
    appendix=[
        ('1. 개발 산출물 예시사례', 15),
        ('2. 스마트화 전환 실사 필요매장', 23),
    ]
))
parts.append(make_page_break())

# 3. 보고내용 요약 페이지
parts.extend(make_summary_page(...))
parts.append(make_page_break())

# 4. 본문 (장·절·항 위계 자동)
parts.extend(make_chapter('Ⅰ', '추진배경 및 목적', sections=[...]))
parts.append(make_page_break())
parts.extend(make_chapter('Ⅱ', '추진계획(안)', sections=[...]))
# ...
```

> 신규 함수 (hwpx_helpers.py 보강 대상):
> - `make_cover_page_full(logo, title, subtitle, date, department, approval_line, doc_meta)` — 풀버전 표지 (결재선 표 포함)
> - `make_toc_page(title, entries, appendix)` — 목차 페이지
> - `make_summary_page(sections, side_boxes)` — 보고내용 요약 페이지
> - `make_chapter(roman, title, sections)` — 장(章) 빌더 (자동 페이지 break)
> - `make_section(arabic, title, items)` — 절(節) 빌더
> - `make_subsection(korean, title, items)` — 목(目) 빌더 (가. 나. 다.)
> - `make_page_break()` — 강제 페이지 분리

## ⚠️ v3.5.0 신규: 풀버전 보고서 4대 함정과 자동 검사

`templates/format_full/standard.hwpx` 의 슬롯 구조에는 v3.4.0 까지 자주 발생하던 4가지 함정이 있었습니다. v3.5.0 부터는 `scripts/build_full.py` 가 이를 자동 검사합니다.

### 함정 ①: 목차 슬롯 위계 위반

양식의 목차 슬롯은 **위치별로 위계가 고정**되어 있습니다.

| 위계 | 슬롯 토큰 | 매핑할 값 |
|------|----------|----------|
| Ⅰ급 대제목 | `목차_항목_001`, `_003`, `_013`, `_021`, `_033`, `_043` | `Ⅰ.`~`Ⅵ.` 시작 |
| 소제목 | 그 외 홀수 토큰 (`_005`, `_007`, ...) | `1.`~`5.` 시작 (들여쓰기 2칸) |
| 페이지번호 | 짝수 토큰 (`_002`, `_004`, ...) | 숫자만 (자동 매핑) |

**위반 시 증상**: Ⅲ장 제목이 일반 본문 서식으로, "1. 항목명" 이 대제목 서식으로 표시.

**자동 검사**: `build_full.py` 가 `check_chapter_slot_hierarchy()` 로 점검.

### 함정 ②: 빈 슬롯으로 인한 점선 끊김

양식은 **고정 슬롯 수** (대제목 6개 + 소제목 20개 = 텍스트 26개) 를 갖습니다. 콘텐츠가 적어 슬롯을 비우면 빈 행이 발생하고 점선·페이지번호 정렬이 깨집니다.

**해결**: 콘텐츠 구성을 양식 슬롯 수에 맞춰 확장.

| 장 | 권장 소제목 수 |
|---|---|
| Ⅰ | 0 (개요만) |
| Ⅱ | 4 |
| Ⅲ | 3 |
| Ⅳ | 5 |
| Ⅴ | 4 |
| Ⅵ | 4 |

**자동 검사**: `build_full.py` 가 빈 슬롯 비율 20% 초과 시 경고.

### 함정 ③: 본문 마커 중복

본문_항목 슬롯은 마커 처리 방식이 두 가지입니다.

| 슬롯 범위 | 마커 방식 | 콘텐츠 작성법 |
|----------|----------|--------------|
| `본문_항목_001` ~ `_009` | paraPr 자동 ○ 마커 | 마커 없이 텍스트만 (`"브이월드 4단계 고도화로..."`) |
| `본문_항목_010` ~ `_012` | 양식 원본에 ◦ 표기 | 콘텐츠에도 직접 표기 (`"  ◦ 실무 부서 교육..."`) |

**위반 시 증상**: `○ ◦ 브이월드 4단계...` 처럼 마커 두 개가 연속 표시.

**자동 검사**: `build_full.py` 가 `check_body_marker_consistency()` 로 점검.

### 함정 ④: 페이지번호 자동 매핑이 hwpx 에 반영 안 됨

v3.4 의 `simulate_pages.py` 는 페이지번호를 **계산만** 하고 hwpx 에 적용하지 않았습니다.

**v3.5.0 핫픽스**: `simulate_pages.py` 에 `--values` + `--apply-to-values` 옵션 추가. `build_full.py` 가 자동으로 호출.

## ⭐ v3.5.0 권장 빌드 워크플로우

기존 5단계(`fill_skeleton` → `fix_namespaces` → `simulate_pages` → 수동 갱신 → `fill_skeleton` 재실행 → `fix_namespaces` → `ensure_body_anchor` → `validate`) 를 한 줄로 단축:

```bash
python3 scripts/build_full.py \
    --values my_values.json \
    --output result.hwpx
```

**자동 처리 항목**
- 입력 검사 (4대 함정 사전 감지)
- 1차 빌드 → 페이지번호 시뮬레이션 → values 자동 갱신 → 2차 빌드
- 후처리 3종 (`fix_namespaces`, `ensure_body_anchor`, `validate`)
- `--strict` 옵션 시 위반 발견하면 빌드 중단
