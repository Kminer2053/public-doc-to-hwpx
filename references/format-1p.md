# 1페이지 보고서 (1p Report) 작성 가이드

> 우수예시: `○○기관 협력사업 실무협의 보고`, `신규 사업 제휴 검토결과 보고`, `○○ 시설 언론보도 현황 및 향후 계획`
> 공공기관 보고서 작성 강의자료 p.15 + 공공기관 우수예시 3건 기반

> **v3.1 기준 빌드 베이스**: `templates/format_1p/standard.hwpx` (맑은 고딕, charPr 21개)가 기본 베이스로 자동 적용됩니다. 사용자가 `--reference` 로 별도 파일을 주면 그 파일이 우선됩니다.

---

## 1. 핵심 철학

> **의사결정자가 30초 안에 핵심을 파악하고, 5분 안에 의사결정할 수 있게 하는 보고서.**

A4 1쪽이라는 강제 제약은 **불필요한 것을 모두 덜어내게 만드는 장치**.

## 2. 표준 골격 (Layout Skeleton)

```
┌──────────────────────────────────────────────────┐
│         - 부제(보고배경 1줄) -                       │  ← 13pt 보통
│       제 목 (대제목, 18pt 굵게, 검정)                  │
│                                                  │
│                          작성자(부서·직책)('YY.MM., ☎)  │  ← 11pt 우측
│  ┌──────────────────────────────────────────┐    │
│  │ [음영 박스] 보고 목적·요지를 1–2줄 두괄식 요약            │ │ │  ← 11pt
│  └──────────────────────────────────────────┘    │
│                                                  │
│ □ 대제목 1 (15pt 굵게)                              │
│   ○ 본문 항목 (12pt)                                │
│   ○ 본문 항목                                      │
│     - 세부 항목 (11pt)                              │
│       * 부가 설명 (10pt 회색)                         │
│                                                  │
│ □ 대제목 2                                        │
│   ○ 본문 항목                                      │
│     - 세부 항목                                    │
│                                                  │
│ □ 대제목 3 (보통 향후계획 또는 결론)                     │
│   ○ 본문 항목                                      │
└──────────────────────────────────────────────────┘
```

## 3. 필수 구성 항목 (4개 — 1쪽 안에 모두 들어가야 함)

| # | 항목명 | 별칭 | 역할 |
|---|--------|------|------|
| 1 | **개요/배경** | 추진배경, 보고배경, 검토배경 | 왜 이 보고를 하는가 |
| 2 | **주요 내용** | 추진경과, 검토결과, 진행현황, 보도내용 | 무엇을 보고하는가 |
| 3 | **세부 사항** | 검토의견, 분석결과, 처리내역 | 어떻게 처리·판단했는가 |
| 4 | **향후 계획** | 조치사항, 후속조치, 결론 | 무엇을 할 것인가 |

> 필수 항목 누락 시 `※ AI 보완 필요` 플래그.
> 4개 모두 채울 콘텐츠가 없으면 풀버전 보고서로 양식 변경 권고.

## 4. 항목별 작성 요령

### 4-1. 제목 + 부제

- **부제**: 검정 13pt, 대시 양 옆 (`- 부제 -`), 가운데 정렬
  - 예) `- ○○공사와 중소기업 상생협력을 위한 -`
- **제목**: 검정 18pt 굵게, 가운데 정렬, 명사형으로 짧게
  - 예) `○○기관 협력사업 실무협의 보고`
- **작성자 정보**: 우측 하단 11pt, 형식 `부서·직책 작성자명('YY.MM., ☎번호)`

### 4-2. 음영 박스 (요약문) — 두괄식 핵심

- 옅은 회색(`#F2F2F2`) 배경, 좌우 패딩
- 11pt, 굵게, 검정
- **1–2줄(약 60–80자) 안에 보고 목적·결론 압축**
- 어조: `~을 보고드림.` / `~ 검토결과를 보고드림.`

예시 (공공기관 우수예시):

> ○○협회 ○○○ 대표가 사장님께 서면으로 제안한 "○○ 노선 신규 테마상품 기획"과 관련하여 검토결과를 보고드림.

> ○○공사와 중소기업 상생협력을 위한 ○○기관과 협의진행과정을 보고드림.

> 언론보도된 ○○ 시설 운영현황과 향후 계획을 보고 드림.

### 4-3. □ 대제목

- 짙은 회색(`#595959`) 사각형 마커 + 띄어쓰기 + 제목
- **3–4개 이내**가 가장 좋음 (5개 넘으면 풀버전 권고)
- 마커 다음 단어 사이 한 칸 띄움
- 명사형 짧게 — `추진배경`, `검토결과`, `향후계획`

### 4-4. ○ 본문 항목

- 본문 12pt
- 한 줄 안에 핵심을 끝낼 것 (45자 이내)
- 의문 들 때마다 `한 줄 한 핵심` 원칙으로 분리

### 4-5. - 세부 항목 / * 부가설명 / ※ 비고

- `-` 본문 보충 (11pt)
- `*` 부가설명 (10pt, 회색)
- `※` 주의·비고 (10pt, 약한 강조)
- 세부 들여쓰기는 한 단계씩만 (2단계 들여쓰기 = ○ → -)

### 4-6. 표 (선택)

1페이지 보고서에 표를 넣을 때:
- 표는 1개로 제한 (2개 이상이면 1쪽 안에 들어가기 어려움)
- 단위 표기 우측 상단 (`(단위: 천원)`)
- 표 헤더 옅은 회색 배경
- 합계행 굵게 + 약한 강조

## 5. 분량 가이드 (페이지 1쪽 강제)

A4 1쪽, 여백 상25·하20·좌25·우25mm 기준 **약 35–40줄**.

| 항목 | 권장 줄 수 |
|------|-----------|
| 부제 | 1 |
| 제목 | 1–2 |
| 작성자 | 1 |
| 음영 박스 | 1–2 |
| 빈 줄 | 1 |
| 대제목 1 + 본문 | 6–9 |
| 대제목 2 + 본문 | 6–9 |
| 대제목 3 + 본문 | 4–6 |
| 향후계획 | 3–5 |
| **합계** | **약 24–35줄** |

> 35줄 초과 시 **압축** 또는 **풀버전 변경**.

## 6. 우수예시 분석 (학습용 표준 패턴)

### 6-1. 결과보고형 (`신규 사업 제휴 검토결과 보고`)

```
□ 오페라 개요          ← 객관 사실 정리
  ○ 공연명·일시·장소·주관 (4개 ○ 항목)

□ 제안(요청)사항        ← 무엇이 제안됐나
  ○ 짧게 1줄

□ 검토결과            ← 핵심 (가장 큰 영역)
  [결론 박스: KTX열차 + 오페라 + 호텔 패키지 기획·판매 가능]
  ○ 주관여행사 ~
  ○ KTX운임은 ~ 12%~16% 할인
  ○ 상품판매는 ~

□ 향후계획            ← 액션 아이템
  ○ ~
```

**패턴 특징**: 검토결과가 메인. 서두는 객관사실, 마무리는 액션.

### 6-2. 진행보고형 (`○○기관 협력사업 실무협의 보고`)

```
□ 추진배경           ← 왜 (2개 ○ 항목)

□ 추진경과           ← 무엇이 있었나 (시간순 2개 미팅)
  ○ (6.15. ○○기관 유선협의 ~)
    - 대표적인 ~
    - 산하기관인 ~
  ○ (6.20. 관련기관 협의예정)
    - 일시/장소 / 참석자

□ 향후계획           ← 무엇을 할 것인가 (단계별 일정)
  ○ MOU 체결 ~ : '12.6월
  ○ MOU 체결        : '12.7월
```

**패턴 특징**: 시간 순서로 구성, 우측 정렬 일정 표기.

### 6-3. 동향보고형 (`○○ 시설 언론보도 현황 및 향후 계획`)

```
□ 언론보도 개요      ← 보도 사실관계 (날짜·매체·요지)
□ ○○ 시설 현황   ← 데이터 표 + 보충 통계
□ 향후 계획         ← 결론
```

**패턴 특징**: 표 1개 포함. 데이터 기반 보고.

## 7. 양식 미준수 위험 신호

빌드 전 다음을 점검:

- [ ] 제목이 한 줄을 넘김 → 단축 필요
- [ ] 음영 박스가 3줄 이상 → 1–2줄로 압축
- [ ] □ 대제목이 5개 이상 → 풀버전 변경 권고
- [ ] ○ 본문 항목이 한 □ 아래 7개 이상 → 그룹화
- [ ] 한 ○ 본문이 2줄 이상 → 분리
- [ ] 표가 2개 이상 → 1개로 통합 또는 풀버전 변경
- [ ] 추정 라인 수 38줄 이상 → 압축 알고리즘 가동

## 8. 빌드 시 hwpx_helpers.py 호출 패턴

```python
from hwpx_helpers import (
    reset_id, make_first_para, make_text_para, make_body_para,
    make_section_bar, make_empty_line, make_cover_banner,
    extract_secpr_and_colpr, NS_DECL
)

# 1. secPr 추출 (1p 보고서용 templates/format_1p/header.xml)
secpr, colpr = extract_secpr_and_colpr('templates/format_1p/reference.hwpx')
reset_id()

parts = [
    "<?xml version='1.0' encoding='UTF-8' standalone='yes' ?>",
    f'<hs:sec {NS_DECL}>',
    make_first_para(secpr, colpr),

    # 부제 + 제목 + 작성자
    make_subtitle('- ○○공사와 중소기업 상생협력을 위한 -'),
    make_main_title('○○기관 협력사업 실무협의 보고'),
    make_author_line('○○○처장 ○○○', "'12.6.", '4315'),
    make_empty_line(),

    # 음영 박스 (요약문)
    make_summary_box(
        '○○공사와 중소기업 상생협력을 위한 ○○기관과 '
        '협의진행과정을 보고드림.'
    ),
    make_empty_line(),

    # 본문: □ → ○ → -
    make_section_bar('□', '추진배경'),
    make_body_para('○', '○○공사와 중소기업과의 ...'),
    make_body_para('○', '판로 개척에 어려운 중소기업의 전시·판매를 통한 ...'),
    make_empty_line(),

    make_section_bar('□', '추진경과'),
    make_body_para('○', '(6. 15. ○○기관 유선협의 : ○○과 담당자 ○○○)'),
    make_body_para('-',  '○○공사에서 중소기업 제품에 대한 ...', indent=1),
    make_body_para('-',  '산하기관인 ㈜○○○○○○○ 소개 및 향후 MOU 등 ...', indent=1),
    # ...

    '</hs:sec>'
]
```

> 새로 추가될 함수 (hwpx_helpers.py 보강 대상):
> - `make_subtitle(text)` — 부제 (`- ~ -` 형식, 가운데 정렬, 13pt)
> - `make_main_title(text)` — 1p 대제목 (18pt 굵게, 가운데, 밑줄)
> - `make_author_line(name, date, tel)` — 우측 하단 작성자 라인 (11pt)
> - `make_summary_box(text)` — 음영 박스 (옅은 회색 배경, 11pt 굵게)
> - `make_body_para(marker, text, indent=0)` — 마커별 들여쓰기 자동
