# 이메일 (Email) 작성 가이드

> 공공기관 보고서 작성 강의자료 p.19
> 출력 형태: **.md 또는 .txt** (사용자 답변: 텍스트로만 - 메일 본문 복붙용)

---

## 1. 핵심 철학

> **내·외부 협업자 사이의 빠른 의사소통.**
> 이메일은 짧고, 핵심을 먼저 말하고, 다음 행동을 명확히.

---

## 2. 표준 골격

```
제목: [부서/사업명] 핵심 키워드 + 동작/요청 (예: [AI혁신처] AX포털 1차 검토 회신 요청)

받는사람: name@example.com
참조:    cc@example.com
숨은참조: bcc@example.com (필요 시)

────────────────────────────
(인사 한 줄)
○○○님 안녕하세요. AI혁신처 박○○입니다.

(결론 1–2줄, 두괄식)
○○ 사업의 1차 검토 결과 회신을 요청드립니다.

(상세 — 개조식)
□ 회신 요청 사항
 - ○○ 자료 검토 의견
 - ○○ 항목 보완 사항
 - 첨부 양식 작성 (별지 1호)

□ 기한 및 방법
 - 기한: 2025. 11. 12. (수) 18:00
 - 방법: 본 메일 회신 + 첨부파일

(마무리)
빠른 회신 부탁드립니다. 감사합니다.

──────
박○○ (○○ ○○○처)
02-0000-0000 / mobile 010-0000-0000
example@○○공사.com
```

---

## 3. 강의자료의 5원칙

> 강의자료 p.19에 명시된 핵심 5가지:

1. **제목이 제일 중요** — 글머리기호 활용, 핵심 키워드로 내용 요약
   - 예) `[AI혁신처] AX포털 v1.0 검토 회신 요청 (~11.12)`
2. **결론부터 간략하게 기술** — 중요 정보는 첨부에 있더라도 본문에 꼭 표기
3. **요청사항은 기한을 표기하며 명확하게 작성** — `~까지 / ~분 안에 / ~날 회신`
4. **첨부파일 파일명도 본문과 연계되도록 적절히 명명**
   - ❌ `1.docx`
   - ✅ `[AX포털] 1차 검토 양식_v1.docx`
5. **참조를 활용하여 업무내용을 충분히 공유** — 관련 부서장 cc

## 4. 입력 필드별 권장 작성

### 4-1. 제목

표준 패턴:

```
[부서/사업명] 동사형 결어 + (기한)
```

예시:

| 좋은 제목 | 나쁜 제목 | 이유 |
|-----------|----------|------|
| [AI혁신처] AX포털 1차 검토 회신 요청 (~11.12) | AX포털 검토 부탁 | 부서·기한 누락, 동사 모호 |
| [로코노미추진단] 12월 콘텐츠 일정 협의 요청 | 12월 일정 | 부서·맥락 누락 |
| [긴급] 매출실적 데이터 누락분 회신 부탁 | 회신 부탁드립니다 | 핵심 누락 |

### 4-2. 받는사람·참조

- **받는사람**: 액션을 취해야 할 사람만 (1–3명 권장)
- **참조**: 정보 공유가 필요한 사람 (부서장·협업 부서)
- **숨은참조**: 외부에 발신 시 내부 보고용 (조심해서 사용)

### 4-3. 본문

```
(인사) — 1줄 ~ 2줄
(결론) — 1–2줄 (이 메일의 핵심)
(상세) — □ 글머리로 정리 (개조식)
(마무리) — 1줄 ~ 2줄
(서명) — 부서·연락처
```

총 분량 권장: **본문 200–500자**. 1,000자 넘으면 1p 보고서로 분리하고 이메일은 안내문 역할로.

---

## 5. 글머리 위계 (체계 A 단순화)

이메일은 깊은 위계 부적합. 최대 2단계까지:

```
□ 대제목
 - 세부 1
 - 세부 2
   * 부가 (정말 필요하면 한 단계만 더)
```

---

## 6. 어조·결문

내부 동료:
- `~하시면 감사하겠습니다`
- `~ 부탁드립니다`
- `~ 회신 부탁드립니다`

외부 협력사·기관:
- `~하여 주시기 바랍니다`
- `~ 협조 요청드립니다`
- `~을 회신 부탁드립니다`

긴급:
- `[긴급]`을 제목 맨 앞에
- 본문에 빨강 강조 금지 (메일 클라이언트 호환성)

## 7. 출력 형태 (사용자 답변 반영)

이메일은 **텍스트 출력**으로만 빌드 (.md 또는 .txt). HWPX 빌드 안 함.

출력 파일 구조:

```markdown
# 이메일 본문

**제목**: [AI혁신처] AX포털 1차 검토 회신 요청 (~11.12)

**받는사람**: kimproduct@○○공사.com
**참조**: leemanager@○○공사.com, parkboss@○○공사.com

---

박○○님 안녕하세요. AI혁신처 박○○입니다.

AX포털 v1.0 1차 검토 결과 회신을 요청드립니다.

□ 회신 요청 사항
 - 사용자 흐름 검토 의견
 - 데이터 보안 항목 보완 사항
 - 별지 1호 양식 작성

□ 기한 및 방법
 - 기한: 2025. 11. 12. (수) 18:00
 - 방법: 본 메일 회신 + 첨부파일

빠른 회신 부탁드립니다. 감사합니다.

---
박○○ (○○ ○○○처)
02-0000-0000 / mobile 010-0000-0000
example@○○공사.com
```

## 8. 양식 미준수 위험 신호

- [ ] 제목이 부서명·기한 없이 두루뭉술함 → 자동 보강
- [ ] 본문이 결론 없이 시작함 (인사 → 배경 → 결론 순) → 두괄식으로 재배열
- [ ] 요청사항에 기한이 없음 → 사용자에게 확인
- [ ] 본문이 1,000자 초과 → 1p 보고서 + 안내 메일로 분리 권고
- [ ] `-적/의/것/들` 군더더기 → 자동 정리

## 9. 빌드 시 처리

이메일은 HWPX가 아니므로 **단순 텍스트 빌더**:

```python
def build_email_md(payload: dict) -> str:
    """
    payload = {
        'subject': '...',
        'to': ['...'],
        'cc': ['...'],
        'bcc': ['...'],  # optional
        'greeting': '...',
        'conclusion': '...',
        'sections': [
            {'title': '회신 요청 사항', 'items': [...]},
            {'title': '기한 및 방법', 'items': [...]},
        ],
        'closing': '...',
        'signature': {
            'name': '...', 'org': '...',
            'tel': '...', 'mobile': '...', 'email': '...',
        }
    }
    """
    lines = []
    lines.append(f"**제목**: {payload['subject']}")
    lines.append('')
    lines.append(f"**받는사람**: {', '.join(payload['to'])}")
    if payload.get('cc'):
        lines.append(f"**참조**: {', '.join(payload['cc'])}")
    if payload.get('bcc'):
        lines.append(f"**숨은참조**: {', '.join(payload['bcc'])}")
    lines.append('')
    lines.append('---')
    lines.append('')
    lines.append(payload['greeting'])
    lines.append('')
    lines.append(payload['conclusion'])
    lines.append('')
    for sec in payload['sections']:
        lines.append(f"□ {sec['title']}")
        for item in sec['items']:
            lines.append(f" - {item}")
        lines.append('')
    lines.append(payload['closing'])
    lines.append('')
    lines.append('---')
    sig = payload['signature']
    lines.append(f"{sig['name']} ({sig['org']})")
    lines.append(f"{sig['tel']} / mobile {sig['mobile']}")
    lines.append(sig['email'])
    return '\n'.join(lines)
```

이메일 양식은 hwpx_helpers를 호출하지 않습니다. layout_optimizer.py만 적용.
