# 버전 업데이트 체크리스트

> 스킬에 변경사항 추가 → 패키징 → 재배포 전에 반드시 확인.
> 매 버전 업데이트마다 갱신해야 하는 5개 위치를 빠뜨리지 않기 위함.

## 1. `SKILL.md` — 4군데 갱신

- [ ] **라인 7 메인 제목**: `# 공공기관 보고서 작성 + HWPX 변환 스킬 (vX.Y.Z)`
- [ ] **변경이력 표** (`| 2026-MM-DD | **X.Y.Z** | ...`): 새 행 추가
- [ ] **디렉토리 구조** (라인 15~50 부근): 새 스크립트/파일 표기 추가
- [ ] **Critical Rules 표** (필요 시): 새 규칙 행 + 마지막 줄 버전 매핑 갱신

## 2. `CHANGES.md` — 사용자 친화적 요약

- [ ] **최신 버전 헤더**: `## 최신 버전: vX.Y.Z (YYYY-MM-DD)`
- [ ] **새 섹션 추가**: `### vX.Y.Z — 한 줄 요약` + 문제/해결/사용법
- [ ] **누적 변경표**: `| vX.Y.Z | 한 줄 요약 |` 행 추가
- [ ] **디렉토리 구조**: 새 파일 표기 추가

## 3. `examples/` — 사용 예시 (필요 시)

- [ ] 새 양식 또는 새 사용법이 도입됐다면 해당 예시 `.json` 추가/갱신
- [ ] 옛 테스트 파일이 루트에 남지 않도록 정리

## 4. `scripts/` — 신규 스크립트 (필요 시)

- [ ] 새 스크립트 파일은 상단 docstring 에 버전 명시 (`v3.6.11 신규`)
- [ ] `fill_skeleton.py` 또는 `build_full.py` 등 통합 진입점에서 import + 호출

## 5. 빌드/검증

- [ ] `examples/` 의 예시 JSON 으로 빌드해서 검증 통과 확인
- [ ] `python3 scripts/validate.py <output.hwpx>` 통과
- [ ] 한글2018 / 한컴뷰어 양 환경에서 시각 검증 (사용자 검증 요청)

## 6. 패키징

- [ ] `PYTHONPATH=/mnt/skills/examples/skill-creator python3 -m scripts.package_skill <skill_path>`
- [ ] 결과 `.skill` 파일 사용자 제공

## 자주 빠뜨리는 항목 TOP 3

1. **`CHANGES.md` 갱신** — SKILL.md 변경이력만 추가하고 CHANGES 는 잊기 쉬움
2. **메인 제목 버전** — 표 안에만 새 버전 추가하고 라인 7 제목은 옛 버전 잔존
3. **`examples/` 정리** — 옛 테스트 JSON 이 루트에 남아 사용자가 헷갈림
